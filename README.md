# 🤖 talking-to-robots-but-better

> A collection of reusable Claude / OpenCode agent configs, prompts, and workflows
> for people who are tired of explaining things to AI *five times*.

---

## 🧠 What is this?

This repo is my personal toolkit for building and using **AI coding agents that actually behave**.

Instead of rewriting prompts every time or fighting your tools, this gives you:

* 🔧 Structured agent configs (`.claude/agents`, `opencode.json`)
* 🧩 Reusable skills & knowledge packs
* 🗂️ Clean project-level rules (`AGENTS.md`)
* ⚡ Prebuilt workflows for real dev tasks

---

## 😤 Why this exists

If you've ever:

* Re-explained the same thing to an LLM 3 times
* Had an agent ignore your instructions
* Lost a good prompt forever
* Thought *“this should be reusable”*

Yeah… same.

This repo is the solution.

---

## 📦 What's inside

```
.claude/
  agents/        → custom subagents (reviewer, builder, etc.)
  skills/        → reusable knowledge modules

prompts/         → structured prompt files

AGENTS.md        → project rules / behavior

opencode.json    → agent + tool configuration
```

---

## ⚙️ What you can do with it

* 🏗️ Spin up new agents instantly
* 🧠 Standardize how your AI behaves across projects
* 🔁 Reuse prompts instead of rewriting them
* 🧪 Experiment with different agent roles
* 🚀 Actually *trust* your AI workflows

---

## 🧩 Example Agents

* **code-reviewer** → strict, opinionated, catches bad patterns
* **builder** → writes clean, production-ready code
* **planner** → breaks features into steps
* **debugger** → finds why things *actually* broke

## 🛠️ How to use

1. Clone the repo:

```bash
git clone https://github.com/your-username/talking-to-robots-but-better
```

2. Copy what you need into your project:

* `.claude/`
* `AGENTS.md`
* `opencode.json`

3. Adjust to your stack (React, backend, etc.)

4. Stop arguing with your AI (as much)
